Microsoft Edge is not supported now. Please use chrome, Firefox or Safiri to run IPython notebook

Overview

The purpose of the data mapping API is to provide a simple and flexible way for programmer to display additional information in a pathway. We map the character of data to the property of glyph. Traditionally, limited information could be displayed in the nodes and arcs only by the their color and opacity. In this package user triggered events, such as mouse-click, are employed to demonstrate more details of the data. This processes should be handled, and then output the information in the user-interested entity, via additional-plotting or Popup windows.

This figure shows the procedure of data mapping

This section will cover:

  • Get ID: provide the identifier what pathway file contains, see ID chapter
  • Change prop: change the graphic properity and add interactive element to pathway, see Prop and Event

Get ID

Note

Before data mapping you should have data annotated to database, like genbank, go or KEGG. Then the connection could be built between data and pathway

The support and attribute to get ID is listed in the table below. All these attributes is belonged to pathway node(entity)

  • In KEGG pathway file, pathway.nodes is used to get all nodes in pathway, each has a id property and ko_id property (may be empty). if you are using pathway for epecifc organism, using ko to get the ko_id for node, and organims code like hsa for organism specific ID. These ids could be used in data mapping.

  • In GPML, every DataNode has id and external_id properties, stand for GPML file’s id and external databases’ id.

  • in Reactome pathway file, each glyph node contains id and external_id properties. id stand for graphic id and external_id stand for external database id.

Implement of Properity Setting

We use mapping method integrate(id_lists, visualize_option_lists) to set glyph’s properity.

mapping method

def integrate(id_lists, visualize_option_lists)
  • id_lists: the list of entity ID designate the target of the visualize option.
  • visualize_option_lists: a list of VisualizeOption instance used for specfic id.

Option(aka. VisualizeOption)

This is a class describe the behavior of a certain entity.

The initialization of a VisualizeOption

def __init__(self,
    default=None,
    over=None,
    click=None,
):
  • All of four arguments receive a list of instances of class InteractiveAction(which has children: Prop, HyperLink, PopUp, Connection)
# subclass of dictionary, provide the changed key-value pair value.
class Prop(InteractiveAction):

# hyper-text link
class HyperLink(InteractiveAction):

# PopUp windows
class PopUp(InteractiveAction):

# HighLight or Additional Edge
class Connection(InteractiveAction)
  • default: the default appearance of a entity. We suggest to set the prop of graphic here (like color, opacity).
  • over, click: the mouse event handlers, which provide the function to demonstrate more info to a certain entity(like plot, model or image).

Next we will introduce the subclass of InteractiveAction.

Prop

Prop class is a subclass of the dictionary containing the changeable value of Entity and Edge. 

# init with changed color
vc = Prop(color=‘color’, bg_color=‘blue’, opacity=0.5, scale=2)

HyperLink

HyperLink class stores a URL to certain page of a website, like T cell receptor signaling pathway, which will redirect you to kegg’s pathway path.

# init with a hyper-text Link
hp = HyperLink(name=“KEGG DB”,
    url=“http://www.kegg.jp/kegg-bin/show_pathway?map=hsa04660&show_description=show”)

example: while you mouse over a node, a popup windows will show the link.

PopUp

PopUp class presents a popup windows, which contain tabs for various informations.

A tab receives the following types of content:

  • Table: a html like table to present detailed informations.
  • Text: a text field.
  • Image: a image field,which could be the chemical structures or the experiment results of a entity
  • graph: a barplot, pieplot, lineplot or scatter implemented by Baidu’s Echarts. Plot chart and read the docs shows how to use this Library.
